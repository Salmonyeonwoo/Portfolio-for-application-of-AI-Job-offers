예측 유지보수: GPU 가속 XGBoost를 활용한 시계열 분류 요약

프로젝트 과정 해석 및 기술 역량 상세 설명 (코드 제외)

프로젝트 개요 및 목표

Backblaze 하드 드라이브 SMART 데이터를 사용하여 고장/정상 상태를 분류하는 예측 유지보수(PdM) 모델을 구축했습니다. **NVIDIA RAPIDS (cuML)**을 활용하여 **XGBoost 모델의 학습 및 테스트 과정**을 **GPU로 가속**하여 대규모 데이터 처리 성능을 최적화했습니다.

RAPIDS cuML GPU XGBoost 클래스 불균형 해소

기술적 도전 및 문제 해결 과정

1. 데이터 로드 및 전처리 가속화

**도전:** 방대한 시계열 데이터셋을 CPU 기반 Pandas로 처리할 때 발생하는 속도 저하 문제를 해결해야 했습니다.

**해결:** **RAPIDS cuDF** 데이터 구조를 사용하여 데이터를 GPU 메모리에 직접 로드하고, 불필요 컬럼 제거 및 결측치 처리 등의 모든 전처리 작업을 GPU 상에서 처리하여 **CPU 대비 월등히 빠른 속도**로 데이터 준비를 완료했습니다.

2. 클래스 불균형 문제 해결

**도전:** 하드 드라이브 고장 데이터(Faulty)가 전체 데이터의 극히 일부만을 차지하는 **심각한 클래스 불균형** 문제를 해결해야 했습니다. 이는 모델이 항상 '정상'으로 예측하여 고장 탐지율(Recall)이 낮아지는 원인이 됩니다.

**해결:** `XGBoost` 모델의 **`scale_pos_weight`** 파라미터를 사용하여 소수 클래스(고장)에 높은 가중치를 부여하는 방식으로 해결했습니다. 이 외에도 **오버샘플링(Oversampling)** 기법을 고려했으나, 훈련 속도와 효율성을 위해 가중치 부여를 주 전략으로 선택했습니다.

3. GPU 가속 학습 및 추론

**cuML**을 사용하여 XGBoost Classifier를 훈련하고, 모델 학습 시 **`tree_method='gpu_hist'`** 옵션을 명시적으로 지정하여 모든 훈련 및 추론 과정을 GPU에서 처리하도록 했습니다. 이로써 대규모 예측 유지보수 환경에 적합한 고속 ML 모델을 구현했습니다.

API 키 및 코드 처리 방안

이 프로젝트는 NVIDIA RAPIDS 라이브러리와 공개 데이터셋에 집중했으며 외부 API Key는 사용하지 않았습니다.

**코드 포메이션 대신:** **cuDF를 사용한 데이터 로드 및 전처리 가속화 과정**, **`scale_pos_weight`를 활용한 클래스 불균형 해소 논리**, 그리고 **GPU 가속을 위한 XGBoost 설정** 등 실무적인 모델 최적화 능력을 중심으로 기술 해석을 진행했습니다.
