Project Overview

✓ Goal: A classification problem to predict the `Label` (0 or 1) for `test.csv`, using data from `A.csv` and `B.csv`.

✓ Core Challenge: Type A and Type B data have different features (columns). This requires building separate preprocessing pipelines and models for each type.

✓ Submission: Run `main.py` (training) on the Dacon server to create models (`*.pkl`), which are then used by `script.py` (inference) for prediction.

Data Structure & Preprocessing

Type A Data (`A.csv`)

Contains many string sequence data (comma-separated) like `A1-1` to `A6-1`.

→ Preprocessing: Used `seq_mean` and `seq_std` functions to extract the mean and standard deviation from each sequence, converting them into numeric features.

Type B Data (`B.csv`)

Contains complex paired sequences, like `B1-1` (value) and `B1-2` (condition).

→ Preprocessing: Used `masked_mean` and `seq_rate` to filter values based on a condition (e.g., '1') and calculate their mean/ratio, creating advanced features.

Core Preprocessing & Feature Engineering

Common Preprocessing

  • Age: Converted string values like '10a', '15a' into numbers (10, 15) using the `convert_age` function.
  • TestDate: Split values like '202201' into 'Year (2022)' and 'Month (01)' using the `split_testdate` function.

Data Type Error Resolution

  • TypeError Fix: `np.fromstring` failed when raw integers were present. Solved by explicitly casting inputs to strings using `str(x)`.
  • ValueError Fix: Resolved `object` dtype errors in LightGBM by explicitly adding all remaining string columns to a `drop_columns` list before training.

Model Training & Saving (`main.py`)

Model A (LGBMClassifier)

1. Merged `A.csv` with `train.csv` on `Test_id`.

2. Preprocessed data using `preprocess_A` & `add_features_A`.

3. Trained LightGBM Classifier on the `Label`.

→ Saved as `model/lgbm_A.pkl`.

Model B (LGBMClassifier)

1. Merged `B.csv` with `train.csv` on `Test_id`.

2. Preprocessed data using `preprocess_B` & `add_features_B`.

3. Trained LightGBM Classifier on the `Label`.

→ Saved as `model/lgbm_B.pkl`.

Inference Pipeline (`script.py`)

1. Dacon server executes `script.py`.

2. Loads the `lgbm_A.pkl` & `lgbm_B.pkl` models created by `main.py`.

3. Applies identical preprocessing to the hidden test data (`A_test.pkl`, `B_test.pkl`).

4. Predicts with each model and generates the final `submission.csv`.
