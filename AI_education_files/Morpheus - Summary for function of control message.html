[Morpheus] ControlMessage 파이프라인 통합 요약

메시지에 메타데이터와 태스크 정보를 주입하여 파이프라인 흐름을 동적으로 제어

Morpheus

프로젝트 개요

**ControlMessage** 객체를 활용하여 파이프라인 내 데이터 흐름을 제어하는 방법을 실습했습니다. 메시지에 **메타데이터(metadata)**와 **태스크(tasks)** 정보를 주입하는 커스텀 스테이지를 개발하여, 메시지별로 맞춤형 처리를 지시하고 다운스트림 스테이지가 해당 정보를 기반으로 동작하게 했습니다.

⚙️ 주요 기술 및 역할

  • 기술: Morpheus `ControlMessage`, `SinglePortStage` 커스텀 개발, `DeserializeStage`.
  • ControlMessage 변환: `FileSourceStage`에서 출력되는 `MessageMeta`를 `DeserializeStage`를 통해 `ControlMessage`로 변환하여 메타데이터/태스크를 추가할 수 있는 상태로 만들었습니다.
  • 커스텀 스테이지: `ControlMessage`를 입력/출력으로 받으며, 메시지에 외부에서 주입된 설정 값을 추가하는 `AddMetadataAndTaskStage`를 개발했습니다.

🛠️ 주요 구현 내용

  • 메타데이터 추가: 메시지에 `priority: high`, `source: api`와 같은 일반적인 키-값 쌍의 메타데이터를 추가했습니다.
  • 태스크 정의: 다운스트림 필터링 스테이지가 사용할 수 있도록 `filter`라는 이름의 태스크와 상세 조건(예: `IP: 192.168.1.100`)을 메시지에 첨부했습니다.
  • 타입 명시: 커스텀 스테이지가 `ControlMessage`만 받도록 `accepted_types`를 명시적으로 정의하여 타입 안정성을 확보했습니다.

📌 주요 코드 로직 (ControlMessage 데이터 추가 추상화)

from morpheus.pipeline import SinglePortStage
from morpheus.messages import ControlMessage

class AddMetadataAndTaskStage(SinglePortStage):
    # ... __init__ 메서드에서 metadata와 tasks 초기화 ...

    def accepted_types(self) -> tuple:
        return (ControlMessage, )

    def on_data(self, message: ControlMessage) -> ControlMessage:
        
        # 헬퍼 함수를 사용하여 ControlMessage에 메타데이터와 태스크를 추가
        if self._metadata:
            for key, value in self._metadata.items():
                message.set_metadata(key, value)
                
        if self._tasks:
            for task_name, task_details in self._tasks.items():
                message.add_task(task_name, task_details)
                
        return message
