Morpheus 커스텀 스테이지 구현: 데이터 변환 및 업데이트 요약

프로젝트 과정 해석 및 기술 역량 상세 설명 (코드 제외)

프로젝트 개요 및 목표

Morpheus 파이프라인의 핵심인 **사용자 정의 스테이지(Custom Stage)**를 구현하고, 이 스테이지 내에서 메시지 페이로드의 **cuDF 데이터**를 직접 수정하는 로직을 개발했습니다. 이는 파이프라인의 특정 지점에서 필요한 비즈니스 로직(데이터 변환, 정규화 등)을 삽입하는 기본 역량을 보여줍니다.

기술적 도전 및 문제 해결 과정

1. 커스텀 스테이지 기본 구조 확립

**도전:** Morpheus의 파이프라인과 원활하게 통합되면서도 커스텀 로직을 수행할 수 있는 스테이지 구조를 만들어야 했습니다.

**해결:** `SinglePortStage`와 `PassThruTypeMixin`을 상속받아 스테이지의 기본 입/출력 구조와 타입 정보를 정의했습니다. 핵심 로직은 `on_data` 메서드 내에 구현하여, 들어온 메시지를 처리한 후 다음 스테이지로 전달하도록 했습니다.

2. GPU 데이터의 안전한 수정 로직

**도전:** GPU 메모리에 상주하는 데이터프레임(cuDF)을 여러 스레드가 동시에 접근하여 수정할 때 발생하는 **경쟁 조건(Race Condition)**을 피하고 안전하게 데이터를 업데이트해야 했습니다.

**해결:** 메시지 객체가 제공하는 **`with message.mutable_dataframe() as df:`** 컨텍스트 관리자를 사용했습니다. 이 구문은 데이터 수정 시 잠금(Lock) 메커니즘을 적용하여 안전성을 확보하며, 개발자는 `df` 객체를 마치 Pandas처럼 사용하여 GPU 데이터를 효율적으로 수정할 수 있었습니다.

3. 데이터 변환 로직 구현

예시로, 사용자 인증 로그 데이터에서 **`user` 컬럼의 값을 대문자(Uppercase)로 일괄 변경**하는 로직을 구현했습니다. 이는 데이터 정규화 및 표준화와 같은 실무 전처리 작업에 즉시 적용 가능한 역량을 입증합니다.

API 키 및 코드 처리 방안

이 프로젝트는 Morpheus의 내부 데이터 처리 로직에 집중했으며 외부 API Key는 사용하지 않았습니다.

**코드 포메이션 대신:** **커스텀 스테이지 설계 방법**, **`mutable_dataframe()`을 사용한 GPU 데이터의 안전한 수정 로직**, 그리고 **경쟁 조건 회피** 등 시스템 안정성 및 개발 능력을 중심으로 기술 해석을 진행했습니다.
