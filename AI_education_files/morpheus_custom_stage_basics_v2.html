Morpheus 커스텀 스테이지 기본 구조 및 메시지 업데이트 요약

프로젝트 과정 해석 및 기술 역량 상세 설명 (코드 제외)

프로젝트 개요 및 목표

Morpheus의 핵심인 **사용자 정의 스테이지(Custom Stage)**를 작성하는 방법을 학습하고, 메시지 메타데이터와 페이로드의 데이터를 직접 **업데이트**하는 커스텀 로직을 구현했습니다. 이는 파이프라인의 유연성과 확장성을 확보하는 기본 역량입니다.

Custom Stage 개발 Mutable DataFrame 데이터 업데이트 로직

기술적 도전 및 문제 해결 과정

1. 커스텀 스테이지의 기본 구조 확립

**해결:** `SinglePortStage`를 상속받아 `name`, `supports_cpp_node`, `on_data` 메서드를 정의하는 기본 구조를 확립했습니다. 초기에는 입력 메시지를 그대로 반환하는 `Passthrough Stage`를 구현하여 데이터의 흐름을 먼저 검증했습니다.

2. GPU 데이터의 안전한 수정 로직 구현

**도전:** GPU 메모리 상의 `cuDF` 데이터를 여러 스레드에서 동시에 접근하여 수정할 때 발생하는 **경쟁 조건(Race Condition)**을 피해야 했습니다.

**해결:** `with message.mutable_dataframe() as df:` 구문을 사용하여, 데이터프레임에 대한 **잠금(Lock) 메커니즘**을 적용했습니다. 이 컨텍스트 관리자 덕분에 GPU 데이터를 안전하게 수정할 수 있었으며, 사용자 인증 로그 데이터에서 `user` 컬럼 값을 **대문자(Uppercase)**로 변경하는 로직을 성공적으로 적용했습니다.

3. 데이터 업데이트의 즉각적인 반영

수정된 데이터가 다음 스테이지로 전달될 때, `ControlMessage` 또는 `MessageMeta`에 변경 사항이 즉각 반영되는 것을 확인했습니다. 이는 데이터 정제 및 변환 로직이 파이프라인에 실시간으로 적용됨을 의미합니다.

API 키 및 코드 처리 방안

이 프로젝트는 Morpheus의 내부 데이터 처리 로직에 집중했으며 외부 API Key는 사용하지 않았습니다.

**코드 포메이션 대신:** **`mutable_dataframe()`을 사용한 안전한 데이터 수정 로직**, **경쟁 조건 회피**, 그리고 **Custom Stage 클래스 설계 방법** 등 GPU 기반 파이프라인 개발 능력을 중심으로 기술 해석을 진행했습니다.
