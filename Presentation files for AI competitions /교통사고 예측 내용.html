프로젝트 개요

✓ **대회 목표:** 제공된 `A.csv`와 `B.csv` 데이터를 학습하여, `test.csv`의 `Label` (0 또는 1)을 예측하는 **분류(Classification)** 문제입니다.

✓ **핵심 과제:** A 타입과 B 타입 데이터는 서로 다른 특징(컬럼)을 가집니다. 따라서 **각 타입에 맞는 별도의 전처리 파이프라인과 모델**을 구축해야 합니다.

✓ **제출 방식:** Dacon 서버에서 `main.py` (학습)를 실행해 모델(`*.pkl`)을 생성하고, `script.py` (추론)가 이 모델을 불러와 예측을 수행합니다.

데이터 구조 및 전처리

A 타입 데이터 (`A.csv`)

`A1-1` ~ `A6-1` 등 쉼표(,)로 구분된 **문자열 시퀀스** 데이터가 다수 포함되어 있습니다.

**→ 전처리:** `seq_mean`, `seq_std` 함수를 사용해 각 시퀀스의 **평균값**과 **표준편차**를 추출하여 숫자형 특징(Feature)으로 변환했습니다.

B 타입 데이터 (`B.csv`)

`B1-1` (값), `B1-2` (조건)처럼 **두 컬럼이 쌍으로** 이루어진 복잡한 시퀀스 데이터가 포함됩니다.

**→ 전처리:** `masked_mean`, `seq_rate` 함수를 사용해 **조건(예: '1')에 맞는 값들만 필터링**하여 평균/비율을 계산하는 등 고급 특징을 추출했습니다.

핵심 전처리 및 특징 공학

공통 전처리

  • **Age (나이):** '10a', '15a' 같은 문자열을 `convert_age` 함수로 숫자 (10, 15)로 변환했습니다.
  • **TestDate (날짜):** '202201' 같은 값을 `split_testdate` 함수로 '연도(2022)'와 '월(01)'로 분리했습니다.

데이터 타입 오류 해결

  • **TypeError 해결:** `seq_mean` 처리 시, 일부 숫자(int)가 포함되어 `np.fromstring` 오류가 발생. `str(x)`로 강제 변환하여 해결했습니다.
  • **ValueError 해결:** 전처리 후 남은 `object` (문자열) 컬럼들을 `drop_columns` 리스트로 명시하여 학습 전 모두 제거했습니다.

모델 학습 및 저장 (`main.py`)

Model A (LGBMClassifier)

1. `A.csv`와 `train.csv`를 `Test_id` 기준으로 병합.

2. `preprocess_A`, `add_features_A` 함수로 전처리.

3. LightGBM 분류 모델로 `Label` 학습.

→ `model/lgbm_A.pkl` 파일로 저장.

Model B (LGBMClassifier)

1. `B.csv`와 `train.csv`를 `Test_id` 기준으로 병합.

2. `preprocess_B`, `add_features_B` 함수로 전처리.

3. LightGBM 분류 모델로 `Label` 학습.

→ `model/lgbm_B.pkl` 파일로 저장.

추론 파이프라인 (`script.py`)

1. Dacon 서버가 `script.py`를 실행.

2. `main.py`가 생성한 `lgbm_A.pkl`, `lgbm_B.pkl` 모델 로드.

3. 숨겨진 테스트 데이터(`A_test.pkl`, `B_test.pkl`)에 대해 학습과 동일한 전처리 수행.

4. 각 모델로 예측 수행 후, `submission.csv` 파일 생성 및 제출.
