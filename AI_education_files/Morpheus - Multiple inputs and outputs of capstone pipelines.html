[Morpheus] 캡스톤 다중 입력/출력 (MIMO) 파이프라인 요약

GPU 텔레메트리 및 네트워크 PCAP 로그를 통합 처리하는 비선형 데이터 파이프라인 설계

Morpheus Network

프로젝트 개요

NVIDIA 교육 과정의 **캡스톤 프로젝트**로, 두 개의 독립적인 소스(GPU 텔레메트리 로그, 네트워크 PCAP 로그)에서 들어오는 데이터를 `RouterStage`를 통해 유형별로 분기시키고, 각 브랜치에서 해당 데이터에 특화된 분석(Triton 추론 포함)을 수행하여 암호화폐 채굴 활동을 탐지하는 **비선형(Non-Linear) 파이프라인**을 완성했습니다.

⚙️ 주요 기술 및 역할

  • 기술: Morpheus `Pipeline` (Graph API), `RouterStage`, `FileSourceStage` (멀티 소스), `TritonInferenceStage`.
  • 비선형 구조 구축: `LinearPipeline` 대신 일반 `Pipeline` 객체를 사용하여 복잡한 그래프 구조를 직접 정의하고, 여러 입력과 출력(MIMO)을 처리하는 로직을 구성했습니다.
  • 동적 라우팅 제어: 메시지에 라우팅 키(예: 'route_a', 'route_b')를 메타데이터로 태깅하는 커스텀 스테이지를 사용하여, `RouterStage`가 메시지를 GPU 브랜치와 PCAP 브랜치로 분기하도록 했습니다.
  • 다중 추론 통합: 각 브랜치(NVSMI/PCAP)에 서로 다른 ML 모델을 `Triton Inference Server`를 통해 연결하여 고성능의 다중 탐지 시스템을 구축했습니다.

🛠️ 주요 구현 내용

  • 멀티 소스 정의: 두 개의 `FileSourceStage`를 정의하고, 각각의 출력을 라우터 이전의 메타데이터 태깅 스테이지로 연결했습니다.
  • RouterStage 연결: `pipeline.add_edge(source, router)` 구문을 사용하여 라우터에 두 개의 입력 소스를 연결했습니다.
  • 분기 및 통합: 라우터의 출력 포트를 사용하여 두 개의 독립적인 처리 브랜치를 정의한 후, 최종적으로 `MonitorStage`에서 두 브랜치의 결과를 다시 하나로 모아 통합 모니터링을 수행했습니다.
