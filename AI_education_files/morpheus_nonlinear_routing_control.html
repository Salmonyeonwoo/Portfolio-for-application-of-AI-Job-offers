프로젝트 과정 해석 및 기술 역량 상세 설명 (코드 제외)

프로젝트 개요 및 목표

Morpheus의 고급 기능인 **비선형 파이프라인 구축**과 `RouterStage`를 활용하여 데이터 메시지를 동적으로 분기하는 로직을 실습했습니다. 이는 단일 데이터 스트림을 조건에 따라 서로 다른 모델이나 분석 로직으로 보내는 데 필수적인 기술입니다.

Morpheus Pipeline RouterStage 조건부 제어

기술적 도전 및 문제 해결 과정

1. 비선형 데이터 흐름 설계

일반적인 `LinearPipeline`은 데이터를 한 방향으로만 처리합니다. 저는 복잡한 분기/결합 구조를 위해 **`Pipeline` 클래스**를 사용하여 메시지 흐름이 자유로운 그래프 구조를 정의했습니다. 이를 통해 파이프라인의 유연성을 확보하고 동적 제어 로직을 삽입할 준비를 마쳤습니다.

2. 조건 기반 동적 라우팅 구현

**도전:** 들어오는 메시지의 데이터(로그인 상태)를 실시간으로 확인하고 '성공'과 '실패' 메시지를 분리해야 했습니다.

**해결:** `RouterStage`를 사용하고, 메시지의 페이로드(cuDF 데이터)를 검사하여 라우팅 키를 반환하는 **커스텀 `key_fn` 함수**를 작성했습니다. 이 함수가 데이터의 `status` 컬럼 값을 기반으로 'success' 또는 'failure' 라우팅 키를 생성하여 메시지를 적절한 다운스트림 브랜치로 분기하도록 논리를 구축했습니다.

3. 파이프라인 정확성 검증

라우팅된 메시지가 올바른 브랜치를 통과했는지 확인하기 위해, 각 브랜치(Success/Failure)의 최종 출력 메시지에 **`processing_branch`** 메타데이터를 추가했습니다. 최종 Sink에서 이 메타데이터를 확인하여 라우팅이 정확하게 이루어졌는지 검증하는 **유닛 테스트 개념**을 적용했습니다.

API 키 및 코드 처리 방안

이 프로젝트는 NVIDIA Morpheus 프레임워크의 핵심 로직을 다루었으므로, 외부에 노출될 수 있는 API Key를 사용하지 않았습니다. 모든 데이터 처리는 GPU 메모리 내에서 폐쇄적이고 안전하게 이루어졌습니다.

**코드 포메이션 대신:** 구체적인 Python 코드는 제외하고, 위에 설명된 **`RouterStage`의 역할, `key_fn` 함수의 논리적 설계, 비선형 구조 선택의 당위성** 등 프로젝트의 핵심 **기술 해석**에 집중했습니다.
