ROS 2/Ubuntu 환경 마스터 가이드

WSL, Conda, ROS 2 빌드 오류를 해결하는 핵심 루틴

핵심 문제 진단: 'Package not found'의 원인

  • **Conda 충돌:** `(llm_agent_env)` 활성화 상태에서 `source install/setup.bash`를 실행하면 Conda 환경이 ROS 경로를 덮어씁니다. **(해결책: `conda deactivate` 후 Source)**
  • **Source 누락:** `colcon build` 후 **새 터미널을 열었거나** `cd` 후 **`source install/setup.bash`** 명령을 잊어버린 경우.
  • **파이썬 빌드 오류:** `setup.py`의 `data_files` 경로 설정이나 `resource` 파일이 누락되어 빌드가 불완전하게 완료됩니다.

⭐ 최종 성공 루틴 (4단계)

ROS 2 패키지 빌드 및 실행을 **단 한 번에** 성공시키는 필수 절차입니다.

  1. 1 Clean 및 리소스 확인

    빌드 환경 오류와 잔여물을 완전히 제거하고, 필수 리소스 파일을 만듭니다.

    # 워크스페이스 루트로 이동 (필수)
cd ~/ros2_study

# 필수 리소스 파일 생성 (Cursor AI 최종 해결책)
mkdir -p src/my_first_package/resource
touch src/my_first_package/resource/my_first_package
touch src/my_first_package/my_first_package_1/__init__.py

# 기존 빌드 결과 완전 삭제
rm -rf install build log
  2. 2 패키지 빌드

    설정 파일(setup.py)을 기반으로 실행 파일을 생성합니다.

    colcon build

    **주의:** 이 단계에서 오류가 없어야 합니다. 오류 발생 시, `package.xml`의 `` 중복을 확인하세요.

  3. 3 환경 로드 (Source)

    ROS 2가 방금 빌드된 패키지를 찾도록 현재 셸에 경로를 등록합니다. **(가장 중요)**

    source install/setup.bash
  4. 4 노드 실행 확인

    `.py` 확장자를 제외하고 `setup.py`에 등록된 이름으로 실행합니다.

    # 등록 확인: my_first_package turtle_cmd_and_pose 가 출력되어야 합니다.
ros2 pkg executables my_first_package

# 최종 실행
ros2 run my_first_package turtle_cmd_and_pose

ROS 코딩 원리 추상화 (Turtle 제어 로직)

우리가 작성한 `turtle_cmd_and_pose.py` 노드가 어떻게 거북이를 자율적으로 제어하는지에 대한 설명입니다.

1. ROS 통신 모델

이 노드는 ROS의 **Publisher/Subscriber (발행/구독)** 모델을 사용합니다. 코드는 하나의 **노드** 역할을 수행합니다.

2. 입력 (Subscriber)

노드는 `/turtle1/pose` 토픽을 **구독**합니다. 이 토픽을 통해 거북이 시뮬레이터(Turtlesim)로부터 거북이의 **현재 X, Y 좌표 및 회전 각도(Theta)**를 실시간으로 입력 받습니다.

3. 제어 로직 (Decision Making)

입력된 위치(`Pose`) 정보가 **X 좌표 8.0**을 초과하는 순간, 노드는 '벽에 부딪힐 위험'을 감지합니다. 이 때, 노드는 주행 상태를 변경합니다.

  • **주행 (Moving):** 평소에는 `Linear.x = 2.0` (직진) 명령을 `/turtle1/cmd_vel` 토픽으로 발행합니다.
  • **회피 (Turning):** X > 8.0이 되면 `Linear.x = 0.0` (정지) 및 `Angular.z = -1.5` (제자리 좌회전) 명령을 발행하여 거북이가 벽에서 멀어지도록 유도합니다.

4. 출력 (Publisher)

노드는 계산된 움직임 명령(`Twist` 메시지, 즉 선속도/각속도)을 `/turtle1/cmd_vel` 토픽으로 **발행**합니다. 이 명령을 Turtlesim이 받아 거북이를 움직이게 됩니다.

Setup.py (핵심 경로 설정)

`package.xml` 복사 오류와 실행 파일 등록을 담당하는 핵심 코드입니다.

# setup.py 파일 내 핵심 섹션 (경로 오류 방지)

# 1. 패키지 리소스 등록 (Cursor AI 해결책)
data_files=[
    ('share/ament_index/resource_index/packages',
        ['resource/' + package_name]),
    ('share/' + package_name, ['package.xml']), # package.xml 복사
    # ...
],

# 2. 실행 파일 등록 (ros2 run 명령 인식)
entry_points={
    'console_scripts': [
        'turtle_cmd_and_pose = my_first_package_1.turtle_cmd_and_pose:main',
    ],
},
