[Morpheus] 커스텀 스테이지 데이터 변환 요약

Mutable DataFrame을 사용하여 cuDF 페이로드의 컬럼 값을 안전하고 효율적으로 변환

Morpheus

프로젝트 개요

Morpheus 파이프라인의 핵심인 **`SinglePortStage`**를 상속받아, 메시지 내부의 데이터 페이로드(cuDF DataFrame)를 수정하는 커스텀 스테이지를 구현했습니다. 이는 데이터 정규화, 문자열 처리, 특정 필드 변환 등 **파이프라인 내의 모든 전처리 및 특징 엔지니어링**을 GPU 상에서 수행할 수 있는 기본 역량입니다.

⚙️ 주요 기술 및 역할

  • 기술: Morpheus `SinglePortStage`, `PassThruTypeMixin`, `MessageMeta`, `ControlMessage`.
  • 데이터 가변성 확보: `with message.mutable_dataframe() as df:` 컨텍스트 관리자를 사용하여 GPU 메모리에 있는 cuDF 데이터에 대한 **안전하고 변경 가능한(Mutable)** 접근을 구현했습니다. 이는 멀티 스레드 환경에서 데이터 무결성을 보장합니다.
  • 데이터 변환 로직: `on_data` 메서드 내에서 cuDF의 문자열 조작 함수(예: `.str.upper()`)를 활용하여 특정 컬럼의 데이터를 일괄적으로 변환하는 GPU 가속 로직을 적용했습니다.

🛠️ 주요 구현 내용 (UpdateMessageStage)

  • 클래스 정의: `UpdateMessageStage`를 정의하고 `on_data` 메서드를 오버라이드하여 실제 데이터 처리 로직을 삽입했습니다.
  • 컬럼 변환 예시: 사용자 인증 로그 데이터에서 'user' 컬럼의 모든 값을 **대문자**로 변경하는 로직을 적용하여 데이터 변환을 시연했습니다.
  • 파이프라인 통합: `FileSource`와 `Deserialize` 이후, `InMemorySink` 이전에 이 커스텀 스테이지를 삽입하여 데이터 처리 흐름을 검증했습니다.

📌 주요 코드 로직 (Mutable DataFrame 접근 추상화)

from morpheus.pipeline import SinglePortStage
from morpheus.messages import MessageMeta 

class UpdateMessageStage(SinglePortStage):
    def on_data(self, message: MessageMeta) -> MessageMeta:
        # mutable_dataframe() 컨텍스트를 통해 cuDF를 안전하게 수정
        with message.mutable_dataframe() as df:
            
            # 예시: 'user' 컬럼의 모든 값을 대문자로 변경
            if 'user' in df.columns:
                df['user'] = df['user'].str.upper()
                
        # 수정된 데이터프레임을 포함하는 메시지를 다음 스테이지로 전달
        return message
