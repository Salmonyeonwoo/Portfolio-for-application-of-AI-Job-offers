1. 프로젝트 개요 및 병렬 처리 목표

Morpheus의 `mrc.core.node.Broadcast` 노드를 활용하여, 들어오는 단일 메시지를 손실 없이 여러 다운스트림 스테이지로 동시에 복제 및 전송하는 `MpBroadcastStage`를 개발했습니다. 이는 실시간 데이터를 탐지, 저장 시각화와 같은 여러 분석 로직으로 병렬 처리하여 처리량(Throughput)을 극대화하는 데 필수적인 기술입니다.

핵심 기술 역량

MRC Broadcast Node Multi-Output Port Definition Parallel Processing Pipeline Integration

2. 기술적 도전 및 해결 과정

도전 1: 다중 출력 포트 정의 및 MRC 로직 통합

일반적인 스테이지와 달리, 메시지 복제와 전송을 위해 3개의 출력 포트를 정의하고 MRC 그래프에 `Broadcast` 노드를 명시적으로 통합해야 했습니다.

✅ 해결: `Stage` 클래스를 상속받아 `_create_ports(1, 3)`을 호출하여 1개 입력, 3개 출력을 정의했습니다. 이후 `_build` 메서드 내에서 `mrc.core.node.Broadcast` 객체를 생성하고, 입력 노드와 출력 포트들을 Broadcast 노드에 연결하는 MRC 로직을 직접 구현했습니다.

도전 2: 복제된 메시지의 동시성 및 무결성 검증

3개 포트로 복제된 메시지가 동기화되어 전달되고 내용이 손상되지 않음을 보장해야 했습니다.

✅ 해결: 3개의 출력 포트 각각에 독립적인 `InMemorySinkStage`를 연결하고, 최종적으로 각 Sink에서 수집된 메시지의 개수와 내용이 완전히 동일함을 확인하여 Broadcast 로직의 무결성을 검증했습니다.
