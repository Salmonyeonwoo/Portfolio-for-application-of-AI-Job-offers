Morpheus 복합 필터링 및 조건부 분석 요약

프로젝트 과정 해석 및 기술 역량 상세 설명 (코드 제외)

프로젝트 개요 및 목표

단일 데이터 스트림에서 여러 개의 독립적인 **`FilterStage`**를 **병렬로 배치**하여 복합적인 필터링을 수행하고, 특정 조건에 부합하는 메시지에 대해 **실패 로그인 횟수 계산**과 같은 조건부 분석 로직을 적용하는 방법을 실습했습니다. 이는 실시간 보안 모니터링 시스템의 기본입니다.

복합 필터링 병렬 연결 조건부 분석 로직

기술적 도전 및 문제 해결 과정

1. 다중 필터의 병렬 연결 구현

**도전:** 하나의 입력 데이터 복사본을 여러 필터 로직으로 동시에 보내야 했으나, 메시지 복제 없이 효율적으로 처리해야 했습니다.

**해결:** `DeserializeStage`의 출력을 **두 개 이상의 독립적인 `FilterStage`**에 동시에 연결하여 메시지 복제 없이 효율적으로 데이터 스트림을 분리했습니다. 예를 들어, 한 필터는 '특정 포트 접근'을, 다른 필터는 '특정 패킷 크기 초과'를 동시에 검사하도록 했습니다.

2. 조건부 분석 커스텀 로직 개발

**핵심 로직:** 필터링을 통과한 메시지에 대해 **`FailedLoginAnalysisStage`**와 같은 커스텀 스테이지를 개발하여 적용했습니다. 이 스테이지는 들어온 `cuDF` 페이로드를 분석하여 `status == 'failure'`인 레코드를 그룹화하고, 사용자별 **실패 로그인 횟수**를 집계하는 보안 분석 지표를 계산했습니다.

3. 다단계 전처리 및 재사용성

복잡한 전처리 과정을 **재사용 가능한 여러 커스텀 스테이지**로 분리하여 파이프라인에 순차적으로 연결했습니다. 이는 파이프라인의 모듈성을 높이고 유지보수를 용이하게 하는 좋은 설계 원칙입니다.

API 키 및 코드 처리 방안

이 프로젝트는 Morpheus의 내부 필터링 및 분석 로직 구현에 집중했으며 외부 API Key는 사용하지 않았습니다.

**코드 포메이션 대신:** **다중 필터 병렬 연결의 필요성**, **실패 로그인 집계와 같은 조건부 분석 로직의 구현 논리**, 그리고 **파이프라인 모듈화 능력**을 중심으로 기술 해석을 진행했습니다.
