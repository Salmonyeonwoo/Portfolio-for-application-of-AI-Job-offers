Riva/TTS 질의응답 (QA) 시스템 구현 요약

프로젝트 과정 해석 및 기술 역량 상세 설명 (코드 제외)

프로젝트 개요 및 목표

Riva ASR과 TTS, 그리고 BERT 기반의 QA 모델을 통합하여 **음성 입력 기반의 질의응답(QA) 시스템**을 구축했습니다. 사용자가 음성으로 질문하면 시스템이 답변을 찾아 음성으로 응답하는 **완전한 Conversational AI 파이프라인**을 구현했습니다.

End-to-End Conversational AI BERT QA 모델 파이프라인 통합

기술적 도전 및 문제 해결 과정

1. Multi-Step 파이프라인 통합 설계

**도전:** 음성, 텍스트, 그리고 다시 음성이라는 이질적인 데이터 타입을 4단계에 걸쳐 오류 없이 연속적으로 처리해야 했습니다.

**해결:** 다음 4단계로 구성된 파이프라인을 설계하고 통합했습니다. 1. **ASR:** 사용자 음성 입력 → 텍스트 변환 (Riva) 2. **QA 모델:** 변환된 텍스트 질문 → 문서(Context) 내에서 답변 텍스트 추출 (BERT) 3. **TTS:** 추출된 답변 텍스트 → 음성 오디오 합성 (Riva) 4. **출력:** 합성된 음성 오디오 재생 각 단계의 입/출력 데이터 형식을 정확하게 맞추어 파이프라인의 견고성을 확보했습니다.

2. Context 기반 답변 추출 능력

BERT 기반 QA 모델을 사용하여, 단순히 질문과 가장 유사한 문장을 반환하는 것이 아니라 **제공된 문서(Context)** 내에서 질문에 대한 **가장 정확한 답변 구간(Span)**을 추출하는 논리를 구현했습니다. 이는 단순 검색을 넘어선 **이해 기반의 답변 생성 능력**을 의미합니다.

API 키 및 코드 처리 방안

이 프로젝트는 자체 배포한 Riva 서버와 로컬에서 실행되는 BERT QA 모델을 사용했으므로, 외부 상용 API Key는 사용하지 않았습니다.

**코드 포메이션 대신:** **ASR/QA/TTS의 4단계 통합 파이프라인 설계**, **음성-텍스트-음성 변환 로직의 역할**, 그리고 **BERT 모델을 이용한 Contextual Answer 추출 능력** 등 시스템 설계 및 통합 능력을 중심으로 기술 해석을 진행했습니다.
