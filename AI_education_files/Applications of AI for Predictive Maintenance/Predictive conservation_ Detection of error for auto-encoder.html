예측 유지보수: 오토인코더 기반 이상 탐지 요약

재구성 오류 임계치를 활용한 딥러닝 기반 비지도 이상 탐지 기술 구현

1. 프로젝트 개요 및 목표

본 프로젝트는 시계열 데이터에서 기존 패턴과 다른 이상 패턴(Anomalies)을 효과적으로 탐지하기 위해 오토인코더(Autoencoder, AE) 딥러닝 모델을 사용했습니다. 목표는 모델이 정상 데이터 학습을 위해 훈련시킨 후, 정상 패턴에서 크게 벗어나는 데이터를 이상(Anomaly)으로 분류하는 비지도 학습(Unsupervised) 기반의 예측 유지보수 기법을 구현하는 것입니다.

Autoencoder 이상 탐지 재구성 오류 GPU 훈련

2. 오토인코더 구조 및 데이터 흐름 (가시화)

오토인코더는 데이터를 압축(인코딩)했다가 다시 복원(디코딩)하는 구조를 가집니다. 이 과정에서 정상 데이터는 잘 복원되지만, 이상 데이터는 복원이 어렵습니다.

입력 데이터 (X)
➡️
인코더 (Encoder)
(데이터 압축)
↔️
잠재 공간 (Latent Space)
(최소 차원)
↔️
디코더 (Decoder)
(데이터 복원)
➡️
출력 데이터 ($\hat{X}$)

이상 탐지 원리: 재구성 오류와 임계치

재구성 오류가 임계치를 초과할 때 이상으로 분류됩니다.

3. 기술적 도전 및 문제 해결 과정

3-1. '정상 데이터'만을 활용한 훈련 데이터셋 구성

도전: 이상 탐지 모델은 데이터 불균형이 심하기 때문에, 기존의 지도 학습 방식으로는 학습이 어렵습니다. 모델이 정상 패턴만을 정확하게 인지하도록 만들어야 했습니다.

해결: 훈련 데이터셋에서 정상(Non-defective) 샘플만을 필터링하여 오토인코더의 입력으로 사용했습니다. 이 방식은 모델이 정상 데이터의 특징만을 압축하고 복원하는 데 특화되도록 강제하여, 이상 데이터 입력 시 복원 정확도를 크게 떨어뜨리게 됩니다.

3-2. 이상 탐지를 위한 재구성 오류(Reconstruction Error) 활용

핵심 논리: 훈련된 오토인코더에 테스트 데이터를 입력하고, 원본 데이터 $X$와 복원된 데이터 $\hat{X}$ 간의 평균 제곱 오차(MSE)를 재구성 오류로 산출했습니다. 이 오류가 **사전에 설정된 임계치**를 초과할 경우, 해당 샘플을 '이상(Anomaly)'으로 최종 분류하는 논리를 구현했습니다.

3-3. GPU를 통한 딥러닝 모델 훈련 최적화

대용량의 시계열 데이터를 기반으로 하는 딥러닝 모델 특성상, 훈련 속도가 중요합니다. TensorFlow/Keras 환경에서 GPU 가속 기능을 활용하여 오토인코더 모델의 훈련 속도를 최적화했습니다. 이는 복잡한 딥러닝 구조도 효율적으로 관리하고 운영할 수 있는 역량을 입증합니다.
