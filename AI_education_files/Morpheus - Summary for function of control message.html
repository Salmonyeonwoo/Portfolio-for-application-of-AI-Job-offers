[Morpheus] ControlMessage 파이프라인 통합 및 활용 요약

메시지에 메타데이터와 태스크 정보를 주입하는 커스텀 스테이지를 통한 동적 흐름 제어

Morpheus

프로젝트 개요

Morpheus의 핵심 제어 객체인 **`ControlMessage`**를 활용하여 파이프라인 내 데이터 흐름을 동적으로 제어하는 방법을 실습했습니다. 메시지에 **메타데이터(metadata)**와 후속 스테이지가 처리해야 할 **태스크(tasks)** 정보를 주입하는 커스텀 스테이지를 개발하여, 메시지별로 맞춤형 처리를 지시하는 로직을 구현했습니다.

⚙️ 주요 기술 및 역할

  • 기술: Morpheus `ControlMessage`, `DeserializeStage`, `AddMetadataAndTaskStage` (커스텀).
  • 메시지 타입 변환: `FileSourceStage`에서 출력되는 `MessageMeta`를 `DeserializeStage`를 통해 **`ControlMessage`**로 변환하여 메타데이터 및 태스크를 추가할 수 있는 상태로 만들었습니다.
  • 커스텀 스테이지 개발: `ControlMessage`를 입력/출력으로 받으며, 초기화 시 전달받은 설정 값(`metadata`, `tasks`)을 메시지에 주입하는 `AddMetadataAndTaskStage`를 개발했습니다.
  • 동적 정보 주입: 메시지에 `priority: high` 메타데이터를 추가하거나, 다운스트림 필터링 스테이지가 사용할 `filter` 태스크와 상세 조건(예: `IP` 주소 필터링)을 첨부했습니다.

🛠️ 주요 구현 내용 (AddMetadataAndTaskStage)

  • ControlMessage 전용: 커스텀 스테이지의 `accepted_types`에 `ControlMessage`를 명시하여, 파이프라인이 정확한 메시지 타입만 처리하도록 유도했습니다.
  • 메타데이터 설정: `message.set_metadata(key, value)` 함수를 사용하여 메시지에 메타데이터를 효율적으로 추가했습니다.
  • 태스크 정의: `message.add_task(task_name, task_details)` 함수를 사용하여 하위 스테이지가 수행해야 할 동적 명령을 메시지에 포함시켰습니다.

📌 주요 코드 로직 (ControlMessage 데이터 추가 추상화)

from morpheus.messages import ControlMessage
from morpheus.pipeline import SinglePortStage

class AddMetadataAndTaskStage(SinglePortStage):
    # ... __init__ 에서 metadata와 tasks를 인스턴스 변수로 저장 ...

    def accepted_types(self) -> tuple:
        return (ControlMessage, )

    def on_data(self, message: ControlMessage) -> ControlMessage:
        # ControlMessage의 set_metadata/add_task 헬퍼 함수 활용
        if self._metadata:
            for key, value in self._metadata.items():
                message.set_metadata(key, value)
                
        if self._tasks:
            for task_name, task_details in self._tasks.items():
                message.add_task(task_name, task_details)
                
        return message
