[Morpheus] 복합 필터링 및 조건부 분석 요약

FilterStage와 커스텀 분석 로직을 활용한 다중 조건 메시지 선별 및 보안 지표 계산

Morpheus

프로젝트 개요

Morpheus 파이프라인에서 **복합적인 조건(Multiple Conditions)**으로 메시지를 필터링하고, 특정 조건에 부합하는 메시지에 대해 **실시간 보안 분석 로직**을 적용하는 방법을 실습했습니다. 이는 단일 데이터 스트림에서 다양한 분석 브랜치를 구축하는 기본 토대입니다.

⚙️ 주요 기술 및 역할

  • 기술: Morpheus `FilterStage`, `SinglePortStage` 커스텀 스테이지, cuDF 데이터 처리.
  • 병렬 복합 필터링: DeserializeStage의 출력을 여러 개의 독립적인 `FilterStage`에 동시에 연결하여, 패킷 크기, 특정 IP 접근 등 **서로 다른 조건**에 맞는 메시지를 효율적으로 병렬 분리했습니다.
  • 조건부 분석 구현: 들어온 메시지의 페이로드(cuDF)를 분석하여 **실패 로그인 횟수**와 같은 보안 지표를 계산하고, 그 결과를 다운스트림으로 전달하는 **커스텀 분석 스테이지**를 구현했습니다.
  • 다단계 전처리: 복잡한 전처리 과정을 재사용 가능한 여러 커스텀 스테이지로 분리하고 순차적으로 연결하여 모듈성을 높였습니다.

🛠️ 주요 구현 내용 (실패 로그인 분석 로직)

  • 데이터 집계: 커스텀 스테이지 내에서 cuDF의 `groupby()` 및 `merge()` 연산을 사용하여 'failure' 상태의 레코드만 필터링하고 사용자별 실패 횟수를 집계했습니다.
  • 메타데이터 태깅: 계산된 'high\_risk\_users' 목록을 `ControlMessage.set_metadata()`를 통해 메시지에 첨부하여 다운스트림 스테이지가 후속 조치를 취할 수 있도록 했습니다.
  • 파이프라인 정확성 검증: 각 필터 및 분석 단계의 출력을 `InMemorySinkStage`로 받아, 계산된 지표 값(`failed_count`)이 정확한지 검증했습니다.

📌 주요 코드 로직 (실패 로그인 분석 추상화)

# ControlMessage를 받아 보안 분석을 수행하는 커스텀 스테이지 (SinglePortStage 상속)
class FailedLoginAnalysisStage(SinglePortStage):
    def on_data(self, message: ControlMessage) -> ControlMessage:
        with message.payload().mutable_dataframe() as df:
            # 'failure' 상태 레코드 필터링 및 사용자별 실패 횟수 집계
            failed_logins = df[df['status'] == 'failure']
            
            if not failed_logins.empty:
                # 실패 횟수 계산 및 원본 DataFrame에 병합
                # (cuDF 연산을 통한 GPU 가속 처리)
                df = df.merge(
                    failed_logins.groupby('user').size().reset_index(name='failed_count'), 
                    on='user', 
                    how='left'
                ).fillna({'failed_count': 0})
                
                # 고위험군 사용자 목록을 메시지 메타데이터로 태깅
                # message.set_metadata("high_risk_users", ...)
                
        return message
