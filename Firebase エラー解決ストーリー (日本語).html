技術的な課題：永続性レイヤーの構築成功ストーリー

1. 課題概要

Streamlit Cloud環境でRAGインデックス（FAISS）を永続的に保存するためにGoogle Cloud Firestore（GCP）を統合する過程で、予期せぬサーバー認証の競合と環境設定の問題に直面しました。

核心問題：Admin SDK初期化の失敗

経験した主なエラーメッセージ：JSON構文エラー、Datastore Modeの競合、無効な秘密鍵、AttrDictの型不一致など、複合的な問題が発生しました。

2. ステップごとの問題解決プロセス

ステップ1：DBモードの競合とプロジェクト変更

Native Modeを使用すべきプロジェクトがDatastore Modeに設定されていたために発生した400エラーを確認しました。新しいFirebaseプロジェクトをNative Modeで作成し、サービスアカウントキーを交換しました。

ステップ2：Secrets JSONパースと検証

Secretsファイルに含まれるJSON文字列の構文エラー（Unicode制御文字、エスケープ文字）とStreamlitのAttrDict型問題を解決するために、入力形式に関わらず、Secretsキーを標準のPython辞書（dict）に強制的に変換するロジックを以下のように実装しました。

適用されたPythonコード (_get_admin_credentials 関数)

def _get_admin_credentials():
    # ... (Secret 存在の確認は省略)
    
    service_account_data = st.secrets["FIREBASE_SERVICE_ACCOUNT_JSON"]
    sa_info = None

    if isinstance(service_account_data, str):
        # 1. 文字列の場合: JSONをロード
        try:
            sa_info = json.loads(service_account_data.strip())
        except json.JSONDecodeError as e:
            return None, f"[Error] JSON 構文エラー: {e}"
    elif hasattr(service_account_data, 'get'):
        # 2. AttrDict (secrets.toml 辞書形式)の場合: dictに変換
        # Streamlit SecretsからAttrDictタイプとして返されるものを処理
        try:
            sa_info = dict(service_account_data) # AttrDictを標準のdictに変換
        except Exception:
             return None, f"[Error] 辞書変換失敗. タイプ: {type(service_account_data)}"
    
    # ... (project_id, private_key 存在の確認は省略)
    return sa_info, None

ステップ3：無効な秘密鍵とAdmin SDK認証

全ての構文エラーを解決した後も発生した「無効な秘密鍵」エラーは、Secretsに保存されたキーのフォーマットがAdmin SDKが要求するRaw Stringフォーマットと正確に一致しなかったためでした。Secretsを単一行文字列形式に整理することで問題を最終的に解決しました。

ステップ4：Firestoreルールと書き込み権限の確保

アプリのデータベース保存機能が動作するように、`rag_indices`コレクションに対する`read, write`ルールを明示的に付与し、最終的にRAGインデックスの永続化機能を有効にしました。
