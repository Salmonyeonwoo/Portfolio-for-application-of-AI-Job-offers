Morpheus Broadcast 노드 구현 및 활용 요약

프로젝트 과정 해석 및 기술 역량 상세 설명 (코드 제외)

프로젝트 개요 및 목표

Morpheus의 핵심인 **`mrc.core.node.Broadcast`**를 활용하여, 들어오는 단일 메시지를 여러 다운스트림 스테이지로 동시에 복제 및 전송하는 **`MpBroadcastStage`**를 개발했습니다. 이는 실시간 데이터를 여러 분석/저장 로직으로 병렬 처리하는 데 사용됩니다.

MRC Broadcast 다중 출력 포트 병렬 처리

기술적 도전 및 문제 해결 과정

1. 다중 출력 포트 정의 및 관리

**도전:** 일반적인 `SinglePortStage`와 달리, 복제된 메시지를 전송하기 위해 3개의 출력 포트를 정의해야 했습니다.

**해결:** `Stage` 클래스를 상속받고 `_create_ports(1, 3)`을 호출하여 명시적으로 **1개의 입력과 3개의 출력 포트**를 정의했습니다. 이를 통해 파이프라인 빌더가 Broadcast 노드를 인식하고 올바르게 연결할 수 있도록 준비했습니다.

2. 메시지 복제 로직 통합

**핵심 로직:** 커스텀 스테이지의 `_build` 메서드 내에서 **`mrc.core.node.Broadcast`** 객체를 생성했습니다. 들어온 입력 노드를 Broadcast 노드에 연결하고, Broadcast 노드의 출력 포트 3개를 각각 다른 다운스트림 노드(예: 3개의 `InMemorySinkStage`)에 연결하는 MRC 로직을 구현했습니다.

**활용:** 이를 통해, 단일 로그 메시지가 '실시간 탐지 모델', '데이터베이스 저장 로직', '시각화 대시보드' 등 세 가지 독립적인 경로로 **동시에** 전송되도록 병렬 처리 능력을 극대화했습니다.

3. 복제 결과 검증

3개의 출력 포트를 연결한 3개의 `InMemorySinkStage`에서 수집된 메시지의 **개수와 내용이 완전히 동일함**을 확인하여, Broadcast 로직이 데이터 손실 없이 완벽하게 작동함을 검증했습니다.

API 키 및 코드 처리 방안

이 프로젝트는 Morpheus의 내부 MRC 노드를 활용하는 방법에 초점을 맞추었으며 외부 API Key를 사용하지 않았습니다.

**코드 포메이션 대신:** `_create_ports(1, 3)`을 통한 다중 출력 정의, `mrc.core.node.Broadcast`를 활용한 **복제 로직 구현 과정**, 그리고 병렬 처리를 통한 효율성 증대라는 **기술적 해석**에 집중했습니다.
