RIVA/TTS Speech AI 서버 배포 및 활용 가이드 요약

NVIDIA Riva Speech AI SDK를 활용한 고성능 gRPC 서비스 인프라 구축

1. 프로젝트 개요 및 목표

본 프로젝트는 NVIDIA Riva Speech AI SDK를 사용하여 **실시간 음성 인식(ASR)** 및 **음성 합성(TTS)** 서비스를 **GPU 가속 환경**에서 안정적으로 배포하고 클라이언트와 통신하는 방법을 실습했습니다. 이는 고성능 대화형 AI 서비스를 위한 인프라 구축 및 gRPC 기반 통신 설계 능력을 입증합니다.

Riva SDK Docker Container gRPC Client/Server GPU Infrastructure

2. GPU 기반 Riva 서비스 배포 단계 (가시화)

Riva ASR/TTS 서버를 GPU 환경에 컨테이너로 배포하고 클라이언트와 통신을 확립하는 주요 단계를 요약했습니다.

Step 1
Docker 환경 설정 및 모델 다운로드
NVIDIA GPU 드라이버 및 Docker 환경을 확인하고, Riva 모델을 사전 학습된 컨테이너 이미지 형태로 다운로드했습니다.
Step 2
Riva 서버 컨테이너 실행
Docker 명령어를 사용하여 GPU 리소스를 컨테이너에 올바르게 할당하고, ASR/TTS 서비스를 백그라운드에서 실행했습니다.
Step 3
gRPC 클라이언트 통신 확립
Python 환경에서 gRPC 스텁을 사용하여 배포된 Riva 서버와 연결하고, TTS 및 ASR 요청을 성공적으로 전송했습니다.

Riva 서버 - 클라이언트 통신 아키텍처

gRPC 프로토콜을 이용한 고성능 클라이언트-서버 상호작용 구조

3. 기술적 도전 및 문제 해결 과정

3-1. GPU 리소스의 Docker 컨테이너 할당 문제 해결

**도전:** Riva는 GPU 가속이 필수적이지만, Docker 환경에서 GPU 리소스를 컨테이너에 올바르게 연결하고 사용하는 것은 복잡한 환경 설정이 필요합니다.

**해결:** `nvidia-container-toolkit` 설정 및 Docker 명령어(`--gpus all` 등)를 정확하게 이해하고 적용하여, Riva 컨테이너가 GPU를 완벽하게 활용할 수 있는 환경을 구축했습니다. 이를 통해 복잡한 환경 의존성 문제를 회피하고 배포의 신속성을 확보했습니다.

3-2. gRPC를 사용한 실시간 스트리밍 통신 구현

Riva는 고성능을 위해 gRPC를 기본 통신 프로토콜로 사용합니다. 특히 ASR은 실시간 음성 스트리밍을 처리해야 하므로, 단방향이 아닌 **양방향 스트리밍(Bidirectional Streaming)** 기능을 클라이언트에서 구현해야 했습니다.

**해결:** Python gRPC 클라이언트에서 `iterator`와 `yield` 구문을 활용하여 오디오 청크를 서버로 연속적으로 전송하고, 동시에 서버로부터 실시간으로 ASR 텍스트 결과물을 수신하는 **스트리밍 클라이언트 로직**을 성공적으로 개발했습니다.
