프로젝트 과정 해석 및 기술 역량 상세 설명 (코드 제외)

프로젝트 개요 및 목표

Riva ASR과 TTS 기능을 활용하여 **대화형(Conversational) 에이전트**를 구축했습니다. 사용자의 음성 명령을 ASR로 받아들이고, **의도 인식(NLU)**을 통해 적절히 응답하는 로직을 통합하여, 음성 인터페이스 기반의 AI 시스템을 완성했습니다.

Conversational AI NLU (Intent Recognition) 음성 상호작용

기술적 도전 및 문제 해결 과정

1. 대화 흐름 결정을 위한 NLU 통합

**도전:** 단순히 음성을 텍스트로 바꾸는 것을 넘어, 사용자의 발화가 '볼륨 조절'인지 '음악 재생'인지 **의도를 정확히 파악**하는 기능이 필요했습니다.

**해결:** ASR의 출력 텍스트를 NLU(자연어 이해) 모델의 입력으로 넣고, NLU의 결과인 **'의도(Intent)'**와 **'개체(Entity)'**를 받아 대화의 다음 단계를 결정하는 로직을 구현했습니다. (예: "음악 재생해 줘" → Intent: PlayMusic, Entity: None)

2. 실시간 응답 파이프라인 구성

음성 입력 → ASR → NLU → 응답 텍스트 생성 → TTS → 음성 출력의 전체 과정을 짧은 지연 시간(Low Latency) 내에 완료하도록 파이프라인을 구성했습니다. 이처럼 **실시간 상호작용**이 필수적인 서비스(음성 비서, 차량 시스템)에 적합한 아키텍처를 설계하는 능력을 입증했습니다.

API 키 및 코드 처리 방안

이 프로젝트 역시 자체 배포된 Riva 서버를 사용했으며, 외부 상용 API Key는 사용하지 않았습니다.

**코드 포메이션 대신:** **NLU를 통한 의도 파악 로직의 중요성**, **음성-텍스트 데이터의 상호 운용성 확보**, 그리고 **실시간 대화 에이전트 설계 아키텍처**에 대한 이해를 중심으로 기술 해석을 진행했습니다.
