Morpheus 데이터 분할 및 복제 로직 요약

프로젝트 과정 해석 및 기술 역량 상세 설명 (코드 제외)

프로젝트 개요 및 목표

들어오는 단일 메시지(`MessageMeta`)의 데이터를 **특정 조건에 따라 분할**하고, 분할된 메시지만을 **별도의 출력 포트**로 보내는 커스텀 스테이지(`FilterSplitStage`)를 구현했습니다. 이는 원본 데이터는 유지하면서 조건에 맞는 일부 데이터만 심층 분석 경로로 보내는 데 사용됩니다.

조건부 분할 Multi-Output Port cuDF Query

기술적 도전 및 문제 해결 과정

1. 다중 출력 포트 정의 및 메시지 생성

**도전:** 하나의 입력에서 '필터링된 데이터'와 '원본 데이터'라는 두 개의 다른 메시지를 내보내야 했습니다.

**해결:** `Stage` 클래스를 상속받아 `_create_ports(1, 2)`를 선언하여 **2개의 출력 포트**를 정의했습니다. `_build` 메서드 내에서 `message.get_data().query()`를 사용하여 `cuDF` 데이터프레임을 필터링하고, 이 필터링된 데이터프레임으로 **`새로운 MessageMeta 객체`**를 생성하여 첫 번째 출력 포트로 전송했습니다.

2. 조건부 분할 로직 구현

**핵심 로직:** `on_data` 메서드에서 사용자 정의 필터 함수(`filter_fn`)를 적용하여 데이터프레임을 쿼리(Query)했습니다. 예를 들어, `data_len > 100`과 같이 특정 조건(길이가 긴 로그)을 만족하는 행만을 추출하여 새로운 메시지를 만들었습니다.

3. 원본 데이터 Pass-Through 구현

두 번째 출력 포트로는 **원본 메시지**(`MessageMeta` 객체)를 변경 없이 그대로 전송하도록 했습니다. 이로써 **"심층 분석 브랜치(필터링됨)"**와 **"표준 처리 브랜치(원본 전체)"**를 동시에 downstream에서 활용할 수 있는 구조를 완성했습니다.

API 키 및 코드 처리 방안

이 프로젝트는 Morpheus의 내부 데이터 처리 로직에 집중했으며 외부 API Key는 사용하지 않았습니다.

**코드 포메이션 대신:** **`MessageMeta`와 `cuDF`를 활용한 메시지 분할 로직**, **`_create_ports`를 사용한 다중 출력 포트 정의**, 그리고 **조건부 분할과 원본 통과를 통한 데이터 복제 논리** 등 고급 파이프라인 제어 능력을 중심으로 기술 해석을 진행했습니다.
