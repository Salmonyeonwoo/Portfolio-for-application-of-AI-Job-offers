[Morpheus] 비선형 파이프라인 및 라우팅 제어 요약

RouterStage와 key_fn을 활용하여 데이터 메시지를 동적으로 분기하는 로직 실습

Morpheus

프로젝트 개요

Morpheus의 `RouterStage`를 활용하여 메시지의 페이로드(cuDF 데이터)를 검사하고, 그 결과에 따라 메시지를 서로 다른 다운스트림 브랜치로 분기하는 **동적 제어 로직**을 구현했습니다. 이는 '성공 로그인' 메시지는 데이터 저장 브랜치로, '실패 로그인' 메시지는 보안 분석 브랜치로 보내는 등 **비선형 파이프라인** 구축에 필수적인 기술입니다.

⚙️ 주요 기술 및 역할

  • 기술: Morpheus `Pipeline` (Graph API), `RouterStage`, `key_fn` (라우팅 함수), MRC Builder.
  • 비선형 구조 구축: `LinearPipeline` 대신 일반 `Pipeline` 객체를 사용하여 메시지 흐름이 분기했다가 다시 결합되는 복잡한 그래프 구조를 정의했습니다.
  • 조건부 라우팅: `RouterStage`에 사용자 정의 함수(`router_key_fn`)를 제공하여, 메시지의 페이로드(예: 'status' 컬럼)를 기반으로 라우팅 키를 반환하고 데이터 스트림을 분기했습니다.

🛠️ 주요 구현 내용 (RouterStage)

  • 라우팅 키 함수 개발: 메시지 데이터 내 'status' 컬럼 값을 확인하여 "success\_route" 또는 "failure\_route"와 같은 라우팅 키를 반환하는 함수를 작성했습니다.
  • 라우터 연결: `RouterStage`의 출력 포트(`router.output_ports[0]`, `[1]`)를 사용하여 성공 처리 싱크와 실패 처리 싱크에 각각 연결하여 분기 로직을 완성했습니다.
  • 메시지 태깅: 라우팅된 메시지에 **어떤 브랜치를 통과했는지** 확인하는 메타데이터(`processing_branch`)를 추가하여 파이프라인의 성공적인 동적 라우팅을 검증했습니다.

📌 주요 코드 로직 (라우팅 함수 및 연결 추상화)

from morpheus.stages.general.router_stage import RouterStage
from morpheus.messages import ControlMessage

# 1. 라우팅 키 함수 정의: 메시지 페이로드 상태를 기반으로 키 반환
def router_key_fn(message: ControlMessage) -> str:
    df = message.payload().get_data()
    # 'status' 컬럼의 첫 번째 값을 기준으로 라우팅 키 결정
    if df['status'].iloc[0] == 'success':
        return "success_route"
    else:
        return "failure_route"
