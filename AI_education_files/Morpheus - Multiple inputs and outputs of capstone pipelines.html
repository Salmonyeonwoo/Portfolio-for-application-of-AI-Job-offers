Morpheus 캡스톤 다중 입력/출력 (MIMO) 파이프라인 요약

이질적인 데이터 소스(GPU 로그, PCAP)를 통합하여 다중 ML 모델로 처리하는 비선형 아키텍처 구현

1. 프로젝트 개요 및 핵심 기술

NVIDIA 교육 과정의 캡스톤 프로젝트로, GPU 텔레메트리(NVSMI)와 네트워크 패킷(PCAP)이라는 두 가지 이질적인 입력 데이터를 동시에 받아 처리하는 MIMO(Multi-Input/Multi-Output) 비선형 파이프라인을 구축했습니다. 이를 통해 암호화폐 채굴과 같은 다중 보안 위협을 실시간으로 탐지했습니다.

핵심 기술 역량

MIMO Architecture RouterStage (Conditional Routing) Triton Inference Integration Metadata Tagging

Morpheus의 MIMO 파이프라인 구조

2. MIMO 파이프라인 흐름 시각화

Input 1: GPU 텔레메트리 (NVSMI)
Input 2: 네트워크 패킷 (PCAP)

태깅 및 단일 스트림으로 통합

RouterStage
(조건부 분기)

데이터 타입(Tag)에 따라 메시지 분기

분기 1: NVSMI 전용 브랜치

ML 모델 1 (GPU 로그 분석) 추론

분기 2: PCAP 전용 브랜치

ML 모델 2 (네트워크 패킷 분석) 추론

MonitorStage를 통한 최종 결과 통합 모니터링

3. 기술적 도전 및 문제 해결 과정

3-1. 이질적인 데이터의 통합 처리

해결: `Pipeline` 객체에 두 개의 독립적인 `FileSourceStage`를 설정하여 멀티-입력을 구현했습니다. 각 소스 뒤에 커스텀 메타데이터 태깅 스테이지를 배치하여 메시지에 'NVSMI' 또는 'PCAP' 태그를 부여함으로써 데이터의 정체성을 명확히 했습니다.

3-2. 다중 추론 및 조건부 라우팅

해결: 태깅된 메타데이터를 기반으로 `RouterStage`를 사용하여 메시지를 두 개의 독립된 TritonInferenceStage (각기 다른 ML 모델 사용)로 분기했습니다. 이를 통해 하나의 파이프라인에서 두 가지 독립적인 탐지 로직을 병렬적으로 실행했습니다.

3-3. 파이프라인 모니터링 및 성능 검증

두 브랜치의 최종 출력을 다시 하나로 모아 `MonitorStage`를 배치했습니다. 이를 통해 시스템의 통합 처리량(throughput)을 지속적으로 측정하고, 다중 위협 탐지 시스템의 성능을 검증했습니다.
