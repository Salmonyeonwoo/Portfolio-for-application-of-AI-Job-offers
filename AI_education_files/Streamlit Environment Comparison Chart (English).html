Streamlit Environment Comparison Analysis

Key Differences Between Local Development and GitHub/Cloud Deployment Environments

✅ Common Points (Code Consistency)

✔ Main Code Structure (Python): The logic, functions, and classes of the Streamlit app are reused without major changes.

✔ UI/UX (Streamlit Widgets): The behavior and appearance of Streamlit widgets like `st.button`, `st.sidebar`, and `st.columns` remain identical.

✔ Library Loading: The method of importing external libraries using the `import` statement is the same.

❌ Key Differences (Environment and Security)

Item Local Environment GitHub/Cloud
API Key Management/Security Direct Configuration
Stored in a .env file or local .streamlit/secrets.toml and loaded via OS environment variables. 		Secrets Injection
Keys should NOT be allowed to be uploaded to Git, due to very strict policy of PII (Personally Identifiable Information) security; they are securely injected via Streamlit Cloud's Secrets management system.
Execution Environment & Address Local Machine/Terminal
Executed via the streamlit run app.py command, accessed at http://localhost:8501. 		Public Website/Cloud
Hosted on Streamlit Cloud connected to the GitHub repository, using a unique HTTPS URL.
Dependency Management Flexible
Uses packages already installed in the local Python environment; fewer constraints from requirements.txt. 		Strict Control ❗
MUST explicitly list all dependencies in requirements.txt; app fails if dependencies are missing.
Data Persistence Local Disk Storage
Conversation history and audio records can be permanently saved in local_db/. 		Volatile
Data saved to the local disk is lost upon session end or redeployment. (Requires external DB for persistence)
File System Access Local directory paths (e.g., C:/Users/...) are accessible. The environment is isolated; direct access to the local disk is impossible.

* For portfolio purposes, you must ensure that API Key files and the local_db/ folder are excluded using .gitignore when uploading code to GitHub.
