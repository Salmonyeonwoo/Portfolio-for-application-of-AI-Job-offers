예측 유지보수: GPU 가속 XGBoost 활용 시계열 분류

NVIDIA RAPIDS를 이용한 고속 머신러닝 모델 구축 및 최적화 프로젝트 요약

1. 프로젝트 개요 및 목표

본 프로젝트는 Backblaze 하드 드라이브의 SMART 데이터를 활용하여 장비의 고장 및 정상 상태를 예측하는 **예측 유지보수(Predictive Maintenance, PdM)** 모델을 구축하는 데 중점을 두었습니다.

핵심 목표는 **NVIDIA RAPIDS (cuML)** 라이브러리를 사용하여 **XGBoost 모델의 학습 및 추론 과정 전체를 GPU로 가속**함으로써, 대규모 시계열 데이터셋에 대한 처리 성능을 극대화하는 것이었습니다.

RAPIDS cuML GPU XGBoost 클래스 불균형 PdM 분류

2. 데이터 처리 및 모델링 흐름 (가시화)

⬇️

3. 기술적 도전 및 문제 해결 과정

3-1. 데이터 로드 및 전처리 가속화

**도전:** 방대한 양의 시계열 데이터를 CPU 기반 Pandas로 처리할 때 발생하는 속도 병목 현상을 해결하고, 데이터 전 과정에서 GPU 활용률을 높여야 했습니다.

**해결:** **RAPIDS cuDF** 라이브러리를 도입하여 데이터를 GPU 메모리에 직접 로드하고, 불필요 컬럼 제거 및 결측치 처리 등 모든 전처리 작업을 GPU 상에서 수행했습니다. 이를 통해 데이터 준비 단계에서부터 **CPU 대비 압도적인 속도 향상**을 달성했습니다.

3-2. 치명적인 클래스 불균형 문제 해결

**도전:** 하드 드라이브 고장 데이터(Faulty, 소수 클래스)가 전체 데이터의 1% 미만을 차지하는 심각한 클래스 불균형이 존재했습니다. 이로 인해 모델이 항상 '정상'으로 예측하는 경향이 발생하여 실제 고장 탐지율(Recall)이 낮아지는 문제가 있었습니다.

원시 데이터 클래스 불균형 예시 (정상 : 고장)

99% (정상)
1% (고장)

**해결:** `XGBoost` 모델의 핵심 하이퍼파라미터인 **`scale_pos_weight`**를 계산하여 소수 클래스(고장)에 높은 가중치를 부여했습니다. 이를 통해 모델이 소수 클래스를 더 중요하게 학습하도록 유도하여, 대규모 오버샘플링 없이도 고장 탐지 성능을 효과적으로 개선할 수 있었습니다.

3-3. GPU 가속 학습 및 추론 최적화

**cuML**의 XGBoost Classifier를 사용하여 모델을 훈련할 때, **`tree_method='gpu_hist'`** 옵션을 명시적으로 설정했습니다. 이 설정을 통해 부스팅 알고리즘의 핵심 계산인 트리 생성 과정을 모두 GPU에서 병렬 처리하도록 하여, 대규모 데이터 환경에 적합한 고속 예측 모델을 완성했습니다.
