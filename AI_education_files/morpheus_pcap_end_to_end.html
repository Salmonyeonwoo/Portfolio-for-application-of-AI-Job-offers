[Morpheus] PCAP 엔드투엔드 분석 파이프라인 요약

네트워크 패킷(PCAP) 데이터를 활용한 암호화폐 채굴 탐지 E2E 파이프라인 구축

1. 프로젝트 개요 및 주요 기술 역량

본 프로젝트는 클라우드 게임 환경에서 암호화폐 채굴(Cryptomining) 비정상 행위를 실시간으로 탐지하기 위해 Morpheus 데이터 파이프라인을 구축한 사례입니다. 핵심은 대규모 네트워크 패킷(PCAP) 데이터를 GPU 가속화된 XGBoost 모델 추론과 결합하여 End-to-End로 처리하는 고성능 솔루션을 구현하는 것입니다.

주요 기술 역량

Morpheus Linear Pipeline RAPIDS cuDF Triton Inference Server 통합 Custom Feature Engineering

Morpheus를 활용한 고속 데이터 처리

2. 파이프라인 구성 및 처리 흐름 (가시화)

데이터 로드부터 최종 탐지 결과 저장까지, 7단계로 구성된 선형(Linear) 파이프라인의 수직 흐름입니다.

1. FileSourceStage
JSON Lines 형식의 원본 PCAP 데이터 파일 스트리밍 로드
Message Type: MessageMeta
⬇️
2. DeserializeStage
데이터를 배치 크기에 맞춰 분할 및 ControlMessage로 변환 (GPU 준비)
Message Type: ControlMessage
⬇️
3. Custom Preprocessing Stage
네트워크 필드 변환, 통계적 특징 추출 등 복잡한 특징 엔지니어링 수행
Message Type: ControlMessage
⬇️
4. TritonInferenceStage
Triton Inference Server에 `abp-pcap-xgb` 모델 추론 요청 (GPU 가속 추론)
Message Type: ControlMessage
⬇️
5. AddClassificationsStage
추론 결과(확률)를 기반으로 최종 이진 분류 라벨(정상/비정상) 추가
Message Type: ControlMessage
⬇️
6. SerializeStage & WriteToFileStage
분석된 결과를 최종 출력 형식으로 재조합하고 디스크에 저장
Message Type: MessageMeta (Serialize), None (Write)

3. 기술적 도전 및 해결 방안

3-1. Triton Inference Server 통합을 통한 추론 최적화

도전: 실시간으로 발생하는 대규모 네트워크 데이터에 대해 지연 시간 없이 ML 모델을 실행해야 했습니다.

해결: `TritonInferenceStage`를 사용하여 GPU 기반의 Triton Inference Server와 연동했습니다. 이는 모델 추론을 파이프라인 내에서 효율적으로 분리하고, GPU 메모리 공유(`use_shared_memory=True`)를 통해 데이터 복사 오버헤드를 최소화하여 고속 추론을 달성했습니다.

3-2. 커스텀 스테이지를 이용한 복잡한 특징 엔지니어링 구현

암호화폐 채굴 탐지 모델은 IP/Port 변환, 통계적 파생 변수 등 복잡하고 도메인 특화된 특징 엔지니어링을 요구합니다.

해결: Python 또는 C++로 작성된 커스텀 스테이지(custom stage)를 파이프라인의 전처리 단계에 삽입하여, 필요한 모든 데이터 변환 및 특징 추출 로직을 GPU(cuDF) 환경에서 직접 수행하도록 구현했습니다. 이로써 전처리 과정의 병목 현상을 제거하고 데이터 준비 과정을 가속화했습니다.
