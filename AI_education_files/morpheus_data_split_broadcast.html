[Morpheus] 데이터 분할 및 복제 로직 요약

커스텀 스테이지를 통한 메시지 페이로드의 조건부 분할 및 MRC Broadcast 활용

Morpheus

프로젝트 개요

단일 입력 메시지(MessageMeta) 내의 데이터를 특정 필터 조건에 따라 분할하고, 분할된 메시지와 원본 메시지를 각각 별도의 출력 포트로 전송하는 **`FilterSplitStage`** 커스텀 스테이지를 구현했습니다. 이는 비선형 파이프라인에서 데이터 복제(Broadcast) 및 조건부 분할(Split)을 수행하는 핵심 기술입니다.

⚙️ 주요 기술 및 역할

  • 기술: Morpheus `Stage` (PassThruTypeMixin 미사용), MRC Builder, `MessageMeta` 객체 생성.
  • 조건부 데이터 분할: `message.get_data().query()`를 사용하여 메시지 페이로드(cuDF)에서 특정 조건(예: 메시지 길이 > 100)을 만족하는 행만을 분리했습니다.
  • 다중 출력 관리: `self._create_ports(1, 2)`를 선언하여 1개의 입력과 2개의 출력을 정의하고, `_build` 메서드에서 **분할된 데이터(filtered)**와 **원본 데이터(pass-through)**를 독립적인 출력 노드로 전송했습니다.
  • 메시지 복제(Broadcast): 단일 메시지를 여러 경로로 전달하는 MRC의 `Broadcast` 기능을 이해하고 활용했습니다.

🛠️ 주요 구현 내용 (FilterSplitStage)

  • 새 메시지 생성: 필터링된 데이터프레임을 사용하여 `MessageMeta(메시지.meta_data, filtered_df)`와 같이 **새로운 `MessageMeta` 객체**를 생성하여 메모리 효율성을 유지했습니다.
  • Pass-Through: 원본 메시지는 변경 없이 다른 출력 포트로 보내져, 다운스트림 스테이지가 필터링된 데이터와 원본 데이터를 동시에 처리할 수 있는 MIMO 구조를 완성했습니다.
