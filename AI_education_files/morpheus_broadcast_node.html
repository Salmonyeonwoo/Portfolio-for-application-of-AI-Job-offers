[Morpheus] Broadcast 노드 활용 요약

단일 입력 메시지를 다중 다운스트림 스테이지로 동시에 복제 및 전송하는 Broadcast 구현

Morpheus

프로젝트 개요

Morpheus 파이프라인 내에서 단일 입력 메시지를 여러 개의 다운스트림 스테이지로 동시에 복제 및 전송하는 **`Broadcast` 노드**의 커스텀 스테이지(`MpBroadcastStage`)를 구현하고 테스트했습니다. 이 기능은 하나의 데이터 소스를 **이상 탐지, 통계 집계, 모니터링** 등 여러 분석 경로로 동시에 보내야 하는 **비선형 파이프라인** 구축에 필수적입니다.

⚙️ 주요 기술 및 역할

  • 기술: Morpheus `Stage`, MRC (Morpheus Runtime Component) `Broadcast`, 다중 출력 포트 정의.
  • 다중 출력 정의: 상속받은 `Stage` 클래스에서 `self._create_ports(1, 3)`을 호출하여 1개의 입력 포트와 **3개의 출력 포트**를 명시적으로 정의했습니다.
  • Broadcast 노드 활용: `_build` 메서드 내에서 MRC의 `Broadcast(builder, "broadcast")`를 생성하여, 들어오는 메시지를 3개의 독립적인 출력 노드로 복제하도록 MRC 그래프를 구성했습니다.

🛠️ 주요 구현 내용 (MpBroadcastStage)

  • MRC 그래프 연결: `builder.make_edge(broadcast, outgoing_N)` 구문을 사용하여 Broadcast 노드와 3개의 다운스트림 노드를 연결했습니다. Broadcast 노드는 여러 엣지를 가질 수 있어 복제 기능을 수행합니다.
  • 기능 검증: Broadcast 스테이지의 각 출력 포트를 각각 다른 `InMemorySinkStage`에 연결하여, 단일 소스 데이터가 세 곳의 싱크에 정확히 전달되는 것을 검증했습니다.

📌 주요 코드 로직 (Broadcast 노드 빌드 추상화)

import mrc.core.operators as ops
from mrc.core.node import Broadcast
from morpheus.pipeline import Stage

class MpBroadcastStage(Stage):
    def __init__(self, c: Config):
        super().__init__(c)
        # 1개의 입력 포트와 3개의 출력 포트 정의
        self._create_ports(1, 3) 
        
    def _build(self, builder: mrc.Builder, input_nodes: list[mrc.SegmentObject]) -> list[mrc.SegmentObject]:
        # 1. Broadcast 노드 생성 및 입력 연결
        broadcast = Broadcast(builder, "broadcast")
        builder.make_edge(input_nodes[0], broadcast)
        
        # 2. 3개의 출력 노드 생성 및 Broadcast와 연결
        outgoing_1 = builder.make_node("outgoing_1", ops.pass_thru())
        outgoing_2 = builder.make_node("outgoing_2", ops.pass_thru())
        outgoing_3 = builder.make_node("outgoing_3", ops.pass_thru())
        
        builder.make_edge(broadcast, outgoing_1)
        builder.make_edge(broadcast, outgoing_2)
        builder.make_edge(broadcast, outgoing_3)
        
        # 3. 스테이지의 출력 포트에 대응하는 노드 반환
        return [outgoing_1, outgoing_2, outgoing_3]
