Morpheus 내장 필터 활용 및 결측치 처리 요약

FilterStage를 활용한 실시간 데이터 품질 관리 및 조건부 흐름 제어

1. 프로젝트 개요 및 목표

Morpheus의 내장 스테이지인 `FilterStage`를 활용하여 데이터프레임 내용에 따라 메시지를 통과시키거나 드롭(Drop)하는 조건부 로직을 실습했습니다. 목표는 실시간 데이터 스트림에서 불필요하거나 품질이 낮은 데이터 (결측치)를 사전에 제거하여 후속 분석 단계의 효율성을 극대화하는 것입니다.

핵심 기술 역량

FilterStage User-Defined Filter Function (filter_fn) Data Cleansing (DropNullColumns) Conditional Data Flow

Morpheus의 데이터 클렌징 과정

2. 실시간 조건부 필터링 시각화

입력 메시지 (잠재적 결측치 포함)
FilterStage (커스텀 `filter_fn` 적용)

필터링 조건: "error_message 컬럼에 유효한 값 포함 여부"

통과 (Pass)

유효 데이터 → 다음 분석 Stage로 전송

제거 (Drop)

결측 데이터 → 파이프라인에서 제거

3. 기술적 도전 및 문제 해결 과정

3-1. 사용자 정의 함수 기반의 실시간 필터링

해결: `FilterStage`에 사용자 정의 필터 함수(`filter_fn`)를 인수로 전달하여, 메시지 페이로드의 `cuDF` 데이터를 검사하도록 구성했습니다. 이 함수를 통해 결측치가 아닌 유효한 값을 가진 레코드만 통과시키는 로직을 구현했습니다.

3-2. 결측치(Null) 컬럼 제거를 통한 데이터 클렌징

파이프라인 초기에 `DropNullColumnsStage`와 같은 내장 스테이지를 통합하여, Null 값이 과도하게 많은 컬럼을 자동으로 제거했습니다. 이는 모델 훈련 전에 데이터의 차원(Dimensionality)을 줄이고 데이터 품질을 높이는 데 기여했습니다.
