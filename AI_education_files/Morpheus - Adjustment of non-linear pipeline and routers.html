[Morpheus] 비선형 파이프라인 및 라우팅 제어 요약

RouterStage를 활용한 데이터 메시지의 동적 분기 및 비선형 파이프라인 구축

Morpheus

프로젝트 개요

Morpheus의 `Pipeline` 클래스와 `RouterStage`를 사용하여 데이터 메시지를 내용 기반으로 동적으로 분기하는 **비선형 데이터 파이프라인**을 구현했습니다. 이를 통해 정상 로그와 비정상 로그를 각기 다른 분석 브랜치로 분리하여 효율적인 처리가 가능함을 입증했습니다.

⚙️ 주요 기술 및 역할

  • 기술: Morpheus `Pipeline` (vs. `LinearPipeline`), `RouterStage`, `key_fn` (라우팅 키 함수).
  • 비선형 구조: `Pipeline` 객체를 사용하여 메시지 흐름이 여러 방향으로 분기했다가 다시 결합되는 복잡한 그래프 구조를 정의했습니다.
  • 동적 라우팅: `RouterStage`에 사용자 정의 함수(`router_key_fn`)를 제공하여, 메시지의 페이로드(cuDF)를 검사하고 그 결과(예: 'success_route', 'failure_route')에 따라 메시지를 분기하는 논리를 구현했습니다.

🛠️ 주요 구현 내용

  • 라우팅 키 함수 개발: 메시지 데이터 내 'status' 컬럼 값을 확인하여 라우팅 키를 반환하는 함수를 작성했습니다.
  • 라우터 연결: `RouterStage`의 여러 출력 포트(output_ports)를 각각 성공 처리 싱크(`success_sink`)와 실패 처리 싱크(`failure_sink`)에 연결하여 데이터 흐름을 명확히 분리했습니다.
  • 메시지 추적: 라우팅 성공 여부를 확인하기 위해, 라우팅된 메시지에 **어떤 브랜치를 통과했는지** 나타내는 메타데이터를 추가하는 로직을 구현했습니다.

📌 주요 코드 로직 (라우팅 함수 및 연결 추상화)

from morpheus.stages.general.router_stage import RouterStage
from morpheus.messages import ControlMessage

# 1. 라우팅 키 함수 정의 (메시지의 페이로드 상태를 기반으로 키 반환)
def router_key_fn(message: ControlMessage) -> str:
    df = message.payload().get_data()
    # 'status' 컬럼의 첫 번째 값이 'success'인지 확인
    if df['status'].iloc[0] == 'success':
        return "success_route"
    else:
        return "failure_route"
