Morpheus 내장 필터 활용 및 결측치 처리 요약

프로젝트 과정 해석 및 기술 역량 상세 설명 (코드 제외)

프로젝트 개요 및 목표

Morpheus의 내장 스테이지인 **`FilterStage`**를 활용하여 데이터프레임 내용에 따라 메시지를 통과시키거나 드롭(Drop)하는 조건부 로직을 실습했습니다. 이는 실시간 데이터 스트림에서 불필요하거나 품질이 낮은 데이터를 사전에 제거하여 후속 분석 단계의 효율성을 극대화하는 데 사용됩니다.

FilterStage 결측치 처리 조건부 데이터 흐름

기술적 도전 및 문제 해결 과정

1. 사용자 정의 함수 기반의 실시간 필터링

**도전:** 파이프라인 실행 중 동적으로 메시지의 내용을 검사하고 필터링하는 메커니즘이 필요했습니다.

**해결:** `FilterStage`에 **사용자 정의 필터 함수(`filter_fn`)**를 인수로 전달하여, 이 함수가 메시지 페이로드의 `cuDF` 데이터를 검사하도록 구성했습니다. 예를 들어, `error_message` 컬럼에 **결측치(Null)가 아닌 유효한 값**을 가진 레코드만 통과시키는 로직을 구현했습니다.

2. 결측치(Null) 컬럼 제거를 통한 데이터 클렌징

파이프라인 초기에 **`DropNullColumnsStage`**와 같은 내장 스테이지를 통합하여, 전체 데이터셋에서 Null 값이 과도하게 많은 컬럼을 자동으로 제거하는 데이터 클렌징 작업을 수행했습니다. 이는 모델 훈련 전에 데이터의 차원(Dimensionality)을 줄이고 품질을 높이는 데 기여했습니다.

3. 파이프라인 통합 및 검증

`FilterStage`를 `DeserializeStage`와 `InMemorySinkStage` 사이에 배치하여, 데이터 흐름의 중간에 실시간으로 데이터 품질 관리 및 선별 작업이 이루어지도록 했습니다. 필터링된 메시지의 개수를 확인하여 로직의 정확성을 검증했습니다.

API 키 및 코드 처리 방안

이 프로젝트는 Morpheus의 내장 전처리 기능을 활용하는 데 집중했으며 외부 API Key는 사용하지 않았습니다.

**코드 포메이션 대신:** **`FilterStage`의 동작 원리**, **사용자 정의 `filter_fn`을 통한 동적 필터링 논리**, 그리고 **파이프라인 중간 단계에서의 데이터 품질 관리**라는 실무적 접근 방식을 중심으로 기술 해석을 진행했습니다.
